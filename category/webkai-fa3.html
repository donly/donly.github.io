    老生常谈：CSS overflow 那些事儿

    css 的 overflow 属性的用法很广泛也很重要，最近在修改 WordPress 和 CMS 的模板的时候就发现有很多地方是因为不正确地使用 overflow 属性而导致跨浏览器使用会出现诡异的渲染结果。

    overflow 属性定义和说明

    overflow 中文即溢出的意思，在 CSS 中则是指当前元素的内容溢出后应该怎么处理。

    overflow 属性值有：

    • visible：默认值，表示当前元素显示的内容不可截断，这表示它可能会从当前容器元素中溢出，跑到盒模型外面。
    • hidden：此值表示的内容是剪辑，也没有滚动用户界面应提供查看剪辑区域以外的内容。
    • scroll：使用滚动条来显示溢出的内容，使用这一值时，它却不管用户是否需要都使用滚动条机制。
    • auto：自动模式，根据需要使用滚动条。
    • inherit：从父元素继承。

    先看一个简单示例，修改全局变量的值

    var message = "Hello, Welcome to magicalboy.com";

    function show() {
    message="magicalboy";
    }
    show();
    alert(message);

    运行结果为弹出窗口警告信息：magicalboy.

    JavaScript(ECMAScript) 的解释程序遇到未声明过的标识符时，用该变量名创建一个全局变量，并将其初始化为指定的值。

    所以message是一个全局变量。show()函数中遇到message，首先在函数里面即当前内存空间去找，找不到再向上在全局空间继续找，如果找到了，那好，为它所指向的内存地址赋新的值"magicalboy"。如果找不到呢，正如前面所说，用该变量名创建一个全局变量，并将其初始化为指定的值。

    JavaScript像Python等其它动态脚本语言一样支持闭包（closure）这一特性，使得函数可以使用函数之外定义的变量。
