    老生常谈：CSS overflow 那些事儿

    css 的 overflow 属性的用法很广泛也很重要，最近在修改 WordPress 和 CMS 的模板的时候就发现有很多地方是因为不正确地使用 overflow 属性而导致跨浏览器使用会出现诡异的渲染结果。

    overflow 属性定义和说明

    overflow 中文即溢出的意思，在 CSS 中则是指当前元素的内容溢出后应该怎么处理。

    overflow 属性值有：

    • visible：默认值，表示当前元素显示的内容不可截断，这表示它可能会从当前容器元素中溢出，跑到盒模型外面。
    • hidden：此值表示的内容是剪辑，也没有滚动用户界面应提供查看剪辑区域以外的内容。
    • scroll：使用滚动条来显示溢出的内容，使用这一值时，它却不管用户是否需要都使用滚动条机制。
    • auto：自动模式，根据需要使用滚动条。
    • inherit：从父元素继承。

    如果你参考 W3school 的 CSS overflow 属性，你会发现它的 overflow 定义说明中表格中的默认值为 auto，这是错误的。请参考 W3C 标准中的 overflow 定义吧。

    跨浏览器问题

    • overflow:scroll 时滚动条位置：Firefox 3.0.10 的滚动条在盒子里面，标准的应该是在框模型外面，更新版本已经更正。

    • IE 8 扩展盒子的bug

      这里有一个介绍得很详细的参考文章

    • 破坏浮动布局

      IE 6, 7 和 8都会扭曲 overflow 默认的 visible 值并将水平的扩展一个盒子一匹配内容(比如图片)。 这对使用浮动列布局的结构非常痛苦，而且单个扩展的列就能够将其它列挤下去并使布局乱掉！

    • IE6 中 overflow:hidden 必须指定宽度和高度才能起作用

    overflow 的妙用

    • 清除浮动
    • 去掉 IE 浏览器的默认滚动条：设置 body 标签中的 overflow:auto 就可以了。
